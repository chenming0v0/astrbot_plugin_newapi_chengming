# AstrBot 新API插件

一个功能强大的 AstrBot 插件，用于从可配置的上游 API 获取账户用量并生成统计报告，支持查询最近 API 调用日志与当前用户信息。

## 功能特性

- 📊 **用量统计**：从可配置的 API 拉取用量数据，按固定时间跨度统计 RPM/TPM/TopN 模型
- 📋 **日志查询**：获取最近 API 调用日志，包含详细的请求信息
- 💳 **额度查询**：查询当前用户的额度信息，包括剩余额度和已消耗额度
- 🔍 **模型状态**：实时监控各模型的调用成功率和响应时间
- 👥 **用户状态**：查看不同用户的调用成功率统计
- 🎨 **灵活展示**：支持合并转发或纯文本方式展示结果
- 💾 **数据持久化**：可选保存原始 JSON 响应到本地文件

## 安装方法

1. 将插件目录放入 AstrBot 的 `plugins` 文件夹中
2. 重启 AstrBot 服务
3. 在 AstrBot 配置中填写必要的 API 信息

## 配置说明

插件需要在 `_conf_schema.json` 中配置以下参数：

| 配置项 | 类型 | 说明 | 默认值 |
|-------|------|------|--------|
| `base_domain` | 文本 | 接口域名（仅域名，含协议） | `https://new.xigua.wiki` |
| `authorization` | 文本 | Authorization 令牌 | 空 |
| `new_api_user` | 文本 | New-Api-User ID | 空 |
| `request_timeout` | 整数 | HTTP 请求超时时间（秒） | 15 |
| `use_forward` | 布尔 | tokens统计命令是否使用合并转发 | true |
| `log_use_forward` | 布尔 | 日志查询是否使用合并转发 | true |
| `user_use_forward` | 布尔 | 查询额度命令是否使用合并转发 | false |
| `show_top_models` | 布尔 | 是否显示调用最多的模型 | true |
| `top_n_models` | 整数 | 显示调用最多的模型数量 | 3 |
| `log_page_size` | 整数 | 日志查询条数 | 20 |

> **注意**：`authorization` 和 `new_api_user` 是必须配置的参数，否则无法正常使用插件功能。

## 使用方法

插件提供以下命令：

### 1. `/tokens统计`

获取指定时间范围内的 API 使用情况统计报告，包含：
- 总使用量 (tokens)
- 总请求次数
- 平均 RPM (每分钟请求数)
- 平均 TPM (每分钟 Token 数)
- 平均响应时间
- 调用最多的 TopN 模型信息

示例：
```
/tokens统计
```

### 2. `/logs`

查询最近 24 小时内的 API 调用日志，包含：
- 日志时间
- 日志类型（消费/错误/其他）
- 模型名称
- 输入/输出 Token 数
- 响应时间
- 掩码后的 IP 地址

示例：
```
/logs
```

### 3. `/查询额度`

查询当前用户的额度信息，包括：
- 用户名
- 当前剩余额度（美元）
- 已消耗额度（美元）

示例：
```
/查询额度
```

### 4. `/模型状态`

查询最近 15 分钟内各模型的调用情况，包含：
- 总请求数
- 成功/失败次数
- 成功率
- 平均响应时间

示例：
```
/模型状态
```

### 5. `/用户状态`

查询最近 15 分钟内各用户的调用情况，包含：
- 总请求数
- 成功/失败次数
- 成功率

示例：
```
/用户状态
```

## 数据持久化

插件会自动将 API 返回的原始 JSON 数据保存到插件目录下的 `data.json` 文件中，以便在 API 请求失败时可以使用本地缓存数据。

## 插件原理

1. **数据获取**：通过 HTTP 请求从配置的 API 端点获取用量数据、日志信息等
2. **数据处理**：对获取的数据进行解析和统计分析，计算关键指标
3. **报告生成**：将统计结果格式化为易读的报告
4. **消息发送**：通过 AstrBot 消息系统将报告发送给用户

## 故障排除

1. **配置错误**：如果收到 "配置缺少 base_domain" 或类似错误，请检查 `_conf_schema.json` 中的配置是否正确
2. **API 访问失败**：如果 API 请求失败，插件会自动尝试使用本地缓存的 `data.json` 文件
3. **数据为空**：如果报告显示 "获取的数据为空"，可能是因为指定时间范围内没有数据或 API 返回格式不兼容

## 注意事项

- 插件使用标准库发起 HTTP 请求，无需安装额外依赖
- 默认统计时间跨度为 25 小时（1500 分钟）
- 为保护隐私，IP 地址会被部分掩码处理
- 敏感信息（如 Authorization）在日志中会被部分掩码处理
